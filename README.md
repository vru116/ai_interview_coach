# Multi-Agent AI Interview Coach

Интеллектуальная система для проведения технических интервью, построенная на базе мультиагентной архитектуры. Система имитирует работу реального интервьюера и скрытого технического эксперта (Observer), который анализирует ответы кандидата в реальном времени.

## Основные возможности
- **Hidden Reflection:** Скрытый внутренний диалог агентов перед генерацией ответа.
- **Fact-Checking:** Автоматическое выявление технических галлюцинаций (проверено на сценарии с Python 4.0).
- **Dynamic Adaptability:** Динамическое изменение сложности вопросов в зависимости от качества ответов кандидата.
- **Structured Feedback:** Генерация детального отчета с грейдом (Junior/Middle/Senior), анализом Hard/Soft skills и персональным Roadmap.

## Инструкция по установке и запуску

### 1. Создание и активация виртуального окружения
Для изоляции зависимостей проекта используйте `venv`:

```bash
# Для Windows:
python -m venv venv
venv\Scripts\activate

# Для macOS/Linux:
python3 -m venv venv
source venv/bin/activate
```

### 2. Установка зависимостей
Установите все необходимые библиотеки (LangChain, MistralAI, Pydantic, Dotenv):
```bash
pip install -r requirements.txt
```

### 3. Настройка API ключа
Создайте в корне проекта файл .env и добавьте ваш ключ Mistral AI (или настройте ключ для другой выбранной вами модели):
```bash
MISTRAL_API_KEY=ваш_ключ_здесь
```

### 4. Запуск интервью
Запустите основной скрипт и следуйте инструкциям в терминале:
code
```bash
python main.py
```



## Описание файлов проекта

*   **`main.py`** — Точка входа в приложение. Содержит логику управления сессией, основной цикл диалога «Вопрос-Ответ» и механизм автоматического сохранения логов в формате JSON по завершении интервью.
*   **`chains.py`** — Описание логики агентов на фреймворке **LangChain**. Включает системные промпты и цепочки для трех ролей:
    *   **Observer**: скрытый эксперт, который анализирует ответ, проверяет факты (fact-checking) и дает инструкции интервьюеру.
    *   **Interviewer**: «лицо» системы, ведет коммуникацию с пользователем и адаптирует вопросы.
    *   **Feedbacker**: аналитик, генерирующий финальный структурированный отчет.
*   **`models.py`** — Pydantic-схемы данных. Используются для реализации `Structured Output`, что гарантирует получение валидного и строго типизированного JSON от агентов.
*   **`requirements.txt`** — Список всех внешних зависимостей и библиотек, необходимых для корректной работы системы.
*   **`interview_log.json`** — Финальный артефакт сессии, содержащий полную историю диалога, «скрытые мысли» агентов (`internal_thoughts`) и итоговый аналитический фидбек.
